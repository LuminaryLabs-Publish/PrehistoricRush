# PrehistoricRush Runner Event DSK Map

**Repository:** `LuminaryLabs-Publish/PrehistoricRush`

**Timestamp:** `2026-07-08T12:09:27-04:00`

## Purpose

Map the current DSK/domain breakdown around the live runner route and identify the smallest next kit boundary that makes the route fixture-readable.

## Current route architecture

```txt
src/runtime.mjs
  -> src/game.js
  -> createEventBus()
  -> createDomainHost({ eventBus })
  -> createTickScheduler({ host, eventBus })
  -> install dino form / pose / material domains
  -> install camera-domain-kit
  -> install hud-domain-kit
  -> expose PrehistoricRushComposition.snapshot()
  -> import runtime-terrain-v6.mjs
  -> startPresentationPass()
```

`runtime-terrain-v6.mjs` remains the active game loop authority:

```txt
main()
  -> shell()
  -> load Three.js
  -> createPhysics()
  -> setup scene/camera/renderer/terrain/raptor/scatter pools
  -> create app state
  -> register keyboard/button input
  -> requestAnimationFrame(loop)
  -> mutate movement, terrain, contacts, pickups, scene, camera, HUD, and renderer inline
  -> expose PrehistoricRushHost.getState()
```

## Domain map

```txt
browser-shell-domain
  owns: DOM mount, body style, fixed canvas host, panel, status, start button

runtime-entry-domain
  owns: `src/runtime.mjs` thin module entry into `src/game.js`

composition-domain
  owns: event bus, domain host, scheduler, installed domain list, composition snapshot

event-history-domain
  owns: event registration, event emission, event snapshot history

dino-domain
  owns: dino form descriptor, pose descriptor, material descriptor, bundle descriptor

camera-domain
  owns: close-third-person camera descriptor candidate

hud-domain
  owns: readability HUD descriptor and `render(snapshot)` helper

legacy-runner-runtime-domain
  owns: live input, movement, terrain, scatter, collision, pickup, scene, raptor, baseline HUD/camera/render

presentation-pass-domain
  owns: current close camera, readable stride, HUD DOM, and extra render frame

runner-source-state-domain
  missing: pure projection from `app.scene`, `app.state`, `app.input`, `app.frame`, and route metadata

runner-event-domain
  missing: `RunnerMovedEvent` generation and `runner.moved` bridge into the existing event bus

dino-event-bridge-domain
  partially present: dino pose domain listens for `runner.moved`; missing live event source and fixture readback

presentation-frame-domain
  missing: camera/HUD/dino request records and bounded journal

host-presentation-snapshot-domain
  missing: `PrehistoricRushHost.getState().presentation`
```

## Boundary rule

Keep these kits repo-local until the DOM-free fixture proves a stable reusable API. Promote only the minimal proven surface later.

## Acceptance requirement

The next implementation should not change visible output. It should add readback only:

```txt
same gameplay
same camera feel
same HUD content
same raptor stride look
same renderer frame
plus fixture-readable records
```
