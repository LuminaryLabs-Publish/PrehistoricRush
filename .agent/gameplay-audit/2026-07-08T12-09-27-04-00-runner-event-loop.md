# PrehistoricRush Runner Event Loop Gameplay Audit

**Repository:** `LuminaryLabs-Publish/PrehistoricRush`

**Timestamp:** `2026-07-08T12:09:27-04:00`

## Current gameplay loop

```txt
menu scene
  -> start button / Enter / Space starts game

game scene
  -> A / Left Arrow sets left input
  -> D / Right Arrow sets right input
  -> W / Up sets boost input
  -> Space sets jump input
  -> loop applies turn, yaw, speed ramp, jump velocity, gravity, position delta, distance, terrain height
  -> terrain chunks rebuild or refresh
  -> Rapier actor transform updates
  -> collision/contact checks can switch to run-over
  -> pickup checks can increment shards
  -> distance can switch to win
  -> jump input is consumed

result scenes
  -> button text changes to Retry or Run Again
  -> Space/click can restart by setting scene to game
```

## Current authority problem

The runner loop mutates gameplay state inline and then presentation code reads the mutated app state.

That is playable, but it cannot answer these fixture questions yet:

```txt
what action happened this frame?
was the action accepted or ignored?
what movement delta was produced?
was a runner.moved event emitted?
which dino pose update came from that event?
which camera and HUD requests were derived from the same source state?
which scene transition was requested and why?
which contact produced run-over or pickup mutation?
```
