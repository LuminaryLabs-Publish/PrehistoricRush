## Source findings

```txt
game.start():
  increments runId
  replaces RunState and InputState
  clears lastTransitionStepId
  calls coreSimulation.resetResolution()
  emits RunStarted and requests scene transition

browser start():
  calls game.start()
  rebuilds current active content
  updates patch focus and primes center
  resets camera follow

not coordinated:
  Core Motion reset
  Core Physics body/request/frame reset
  articulated motion/dynamics reset
  patch-controller reset
  Worker request generation/cancellation
  active-content generation reset
  renderer time reset policy
  coherent public reset receipt
  first-visible-new-run acknowledgement
```

`Enter` calls browser `start()` unconditionally, so the reset path is not limited to menu, run-over or win states.
