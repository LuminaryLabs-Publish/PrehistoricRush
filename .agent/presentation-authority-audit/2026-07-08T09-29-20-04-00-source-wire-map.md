# PrehistoricRush Presentation Frame Source Wire Map

**Timestamp:** `2026-07-08T09:29:20-04:00`

## Goal

Turn the existing presentation-frame acceptance ledger into an implementation-ready source wire map.

This is still a documentation-only pass. No runtime source files were changed.

## Current source authority read

```txt
index.html
  -> src/runtime.mjs
  -> src/game.js
  -> domain runtime scaffold
  -> dino/camera/HUD domain kits
  -> await import("./runtime-terrain-v6.mjs")
  -> legacy visual runner owns live movement, contact, camera, HUD, renderer frame, and host state
```

`src/game.js` is the correct bridge location for the new presentation contract because it already owns the repo-local event bus, domain host, dino domains, camera domain, HUD domain, `PrehistoricRushComposition.snapshot()`, and the current presentation pass.

`src/runtime-terrain-v6.mjs` is the correct live-source location because it owns the current `app.state`, scene loop, input mutation, movement integration, contact decisions, scene transitions, and `PrehistoricRushHost.getState()`.

## Source wire points

### 1. Runner source snapshot

Add this next to the live state access in `src/game.js` presentation code, not in the renderer.

```txt
source: PrehistoricRushHost.app.state
consumer: presentation contract layer
output: RunnerSourceState
```

Required fields:

```txt
scene
frame
time
x
y
z
jumpY
vy
grounded
yaw
turn
speed
distance
best
shards
input.left
input.right
input.boost
input.jump
terrainChunkCount
rapierEnabled
```

Rules:

```txt
- Values must be plain JSON.
- Values must be rounded where useful for fixture stability.
- The snapshot must not require DOM, WebGL, renderer frame timing, or a Rapier step.
- The snapshot must preserve the current app.state shape.
```

### 6. Presentation frame record

Collect the bridge outputs into one record.

Target output:

```txt
PresentationFrameRecord
```

Required fields:

```txt
frame
runnerSourceState
runnerMovedEvent
dinoPoseFrame
cameraFrameRequest
hudFrameRequest

```

Rules:

```txt
- Record at most one presentation frame per browser frame.
- Keep a bounded in-memory journal.
- Expose latest frame and recent frames through PrehistoricRushHost.getState().presentation.
```
