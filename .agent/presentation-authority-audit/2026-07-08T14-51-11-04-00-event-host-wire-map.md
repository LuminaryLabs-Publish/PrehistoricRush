# PrehistoricRush Presentation Event Host Wire Map

**Timestamp:** `2026-07-08T14:51:11-04:00`

## Goal

Define the exact host-facing authority cutline for the next implementation pass.

The next implementation should prove that current visual runner behavior can be observed as deterministic records before it is extracted into shared kits.

## Existing authority surfaces

```txt
PrehistoricRushComposition
  source: src/game.js
  owns: installed domain snapshots and event-bus history
  problem: not connected to live runner source facts

PrehistoricRushHost
  source: runtime-terrain-v6.mjs
  owns: app reference and getState()
  problem: exposes scene/runner/physics/terrain/renderer, but not presentation records

dino-pose-domain-kit
  source: src/domains/dino/dino-pose-domain-kit.js
  owns: runner.moved -> dino.pose.changed
  problem: no live runner.moved event reaches it

camera-domain-kit
  source: src/domains/camera/camera-domain-kit.js
  owns: close-third-person-v1 descriptor
  problem: descriptor is not converted into camera frame requests

hud-domain-kit
  source: src/domains/hud/hud-domain-kit.js
  owns: readability-hud-v1 descriptor and render(snapshot)
  problem: DOM HUD renderer does not expose a descriptor request/readback pair
```

## Wire map

```txt
src/game.js
  imports from src/presentation/*
  owns presentationJournal
  startPresentationPass()
    reads host.app
    snapshots RunnerSourceState
    records camera/hud/dino requests
    appends PresentationFrameRecord

runtime-terrain-v6.mjs
  imports or receives presentation helpers
  loop()
    snapshots before/after movement
    emits runner.moved through PrehistoricRushComposition.eventBus
    records contact and scene dispatch facts before mutation
  PrehistoricRushHost.getState()
    adds presentation snapshot
```

## Acceptance matrix

```txt
accepted:
  runner moved in game scene
  dino pose bridge consumed runner.moved
  camera request projected from camera descriptor
  HUD request projected from HUD descriptor
  hazard contact records run-over scene dispatch
  pickup contact records shard collection
  distance goal records win dispatch
  host presentation snapshot exposes bounded recent frames

rejected / no-op:
  runner source in menu scene does not emit movement
  zero-delta frame does not emit runner.moved
  airborne jump records rejection reason
  no contact records contact.none
  unchanged scene records scene.unchanged
```

## Stop condition

The next implementation stops when the fixture can replay the event-host wire chain and the browser host can expose the same shape through `PrehistoricRushHost.getState().presentation` without changing visuals.
